2. FLUXOS DE NEGÓCIO
================================================================================

2.1 FLUXO 1: REQUISIÇÃO → FACTURA → RECIBO (Padrão)
┌─────────────────────────────────────────────────────────────┐
│ 1. Cliente solicita serviço aduaneiro                       │
│ 2. Cria-se REQUISIÇÃO DE FUNDOS (Fatura Proforma)          │
│    - Estado: Pendente                                       │
│    - Totalizações: Subtotal, IVA, Retenção                 │
│ 3. Cliente aprova e paga parcialmente (opcional)            │
│    - Estado: Paga Parcialmente                              │
│ 4. Cria-se FACTURA FINAL a partir da Requisição            │
│    - Estado: Pendente                                       │
│    - Herda dados da Requisição                              │
│ 5. Cliente paga a Factura                                   │
│    - Cria-se RECIBO DE PAGAMENTO                            │
│    - Atualiza saldo da Factura                              │
│ 6. Documentos com PDFs profissionais e acesso por email     │
└─────────────────────────────────────────────────────────────┘

2.2 FLUXO 2: FACTURA-RECIBO (Venda a Pronto Pagamento)
┌─────────────────────────────────────────────────────────────┐
│ 1. Cliente solicita serviço aduaneiro com pagamento pronto  │
│ 2. Cria-se FACTURA-RECIBO                                  │
│    - Estado: Paga (automático)                              │
│    - Documento único que prova facturação + recebimento     │
│ 3. Sistema vincula automaticamente a Factura e Recibo       │
│ 4. Geração de PDF profissional                              │
│ 5. Envio por email com anexo                                │
└─────────────────────────────────────────────────────────────┘

2.3 FLUXO 3: AJUSTES (Notas de Crédito e Débito)
┌─────────────────────────────────────────────────────────────┐
│ NOTA DE CRÉDITO (Desconto/Devolução)                       │
│ 1. Cria-se Nota de Crédito (Estado: Pendente)              │
│ 2. Aguarda aprovação de nível superior                      │
│ 3. Ao aprovar: Atualiza saldo do cliente (+)               │
│ 4. Histórico com criador e aprovador registrado             │
│                                                              │
│ NOTA DE DÉBITO (Cobranças Adicionais)                      │
│ 1. Cria-se Nota de Débito (Estado: Pendente)               │
│ 2. Aguarda aprovação de nível superior                      │
│ 3. Ao aprovar: Atualiza saldo do cliente (-)               │
│ 4. Histórico com criador e aprovador registrado             │
└─────────────────────────────────────────────────────────────┘

3. SEGURANÇA E CONTROLE DE ACESSO
================================================================================

3.1 DECORADORES DE PERMISSÃO
✅ @requer_sessao_ativa
   - Valida sessão ativa do utilizador
   - Bloqueia acesso não autenticado

✅ @requer_escrita_financeira
   - Bloqueia acesso de AUDITORES (apenas leitura)
   - Permite: Administrador, Despachante Oficial, Operador Financeiro
   - Mensagem clara ao tentar escrever sem permissão

✅ _user_tem_acesso_total(request)
   - Verifica se utilizador é Administrador
   - Verifica permissões especiais via usuário.permissoes
   - Retorna True apenas para bypass de scoping

✅ _pode_escrever(request)
   - Validação específica para operações financeiras
   - Rejeita auditores com mensagem personalizada
   - Redirectiona para página anterior com erro

3.2 FILTRAGEM POR ESCOPO
✅ _get_user_cliente_filter()
   - Admin: vê TODOS os clientes
   - Despachante: vê apenas seus clientes
   - Operador: vê apenas clientes de sua banca

✅ _get_object_or_404_com_scope()
   - Validação de propriedade do documento
   - Impede acesso a documentos de outros utilizadores
   - Admin pode ver tudo

3.3 VALIDAÇÕES POR ESTADO
✅ Requisição de Fundos
   - Não permite adicionar linhas a requisição anulada
   - Não permite cancelar requisição já anulada

✅ Factura
   - Não permite cancelar factura paga ou já cancelada
   - Apenas "Pendente" e "Parcialmente Paga" podem ser canceladas

✅ Recibo
   - Não permite cancelar recibo já cancelado
   - Apenas Admin pode cancelar

✅ Notas (Crédito/Débito)
   - Apenas criador ou Admin pode aprovar
   - Estados: Pendente → Aprovada/Rejeitada → Cancelada

4.3 CÁLCULOS FINANCEIROS
✅ Requisição de Fundos
   - Subtotal = soma de todos os itens
   - IVA = 14% sobre honorários do despachante
   - Retenção = 6.5% sobre honorários
   - Total = Subtotal + IVA - Retenção
   - Recalculação automática ao editar linhas

✅ Saldos
   - Saldo pendente = Total - Valor Pago
   - Atualização automática ao registrar pagamentos
   - Rastreamento de débito/crédito por cliente

✅ Conversão de Valores
   - Valor extenso em Kwanzas (texto escrito)
   - Formatação de moeda com separadores
   - Suporte a múltiplas moedas (referência)

4.4 RASTREABILIDADE E AUDITORIA
